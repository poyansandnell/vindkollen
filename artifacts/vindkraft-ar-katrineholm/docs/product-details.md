# Product details

Full behavioral detail behind the "Product" bullets in `replit.md`, for `artifacts/vindkraft-ar-katrineholm`.

- Camera-based AR view showing 29 wind turbines at true bearing/distance from the user, each labeled with name + distance.
- Blinking red aviation obstruction lights on turbines at night (22:00–06:00).
- Optional procedurally generated wind sound (Web Audio, no audio files), always routed through the device's main speaker (never the earpiece) even on iOS Safari.
- Live "🔊 Beräknad ljudnivå" dBA panel estimating sound exposure from turbine distance. The wind sound's actual playback volume scales continuously with this same dBA estimate (`soundLevel.ts`'s `dbaToGain`), instead of a plain on/off.
- Explicit always-visible "🔊 Ljud ute / 🔈 Ljud inne" toggle (top controls) that the user controls directly — always starts on "Ljud ute" at app launch. This toggle (not the camera-based sky heuristic) drives both the displayed dBA number (via `applyIndoorAttenuation`, a -35dB subtraction before the panel's own `dbaToGain`) and the actual wind sound gain — but the two use separate mechanisms by design: the panel number goes through `applyIndoorAttenuation` → `dbaToGain`, while the actual playback gain is computed as `applyIndoorGain(dbaToGain(smoothedOutdoorDba), indoor)`, i.e. `dbaToGain` on the *unattenuated* outdoor dBA, then a direct linear multiplier (`INDOOR_SOUND_GAIN_MULTIPLIER = 10**(-35/20)`) applied afterward. This avoids a bug where subtracting 35dB and re-running it through `dbaToGain`'s floor/ceiling clamp silently produced *zero* audible change whenever the outdoor level was already near the audibility floor (common at real Katrineholm-to-turbine GPS distances) — both "ute" and "inne" clamped to the same near-zero gain. The multiplier approach guarantees a consistent, always-perceptible ~35dB-equivalent drop regardless of the absolute outdoor level.
- "🌗 Skuggflimmer" (shadow flicker) mode simulating blade-shadow flicker, only active in "Aktuell sol"/"Låg sol" sun modes, with an info tooltip explaining the effect.
- "📸 Fotomontage": captures a composite photo (camera + AR overlay + watermark + disclaimer) with Spara/Ta ny bild/Dela (Web Share API) actions.
- "🌬️ Infraljud"-monitor: always-visible green/yellow/red badge + expandable panel estimating overall noise/infrasound impact (a distinct indicator from the dBA panel above) from distance, number of contributing turbines, wind direction (if available), and how long the user has been on-site, with a calm Swedish disclaimer emphasizing it is an estimated indicator, not a medical measurement (never claims guaranteed harm).
- Map view (SVG-based) showing all turbines and the user's position.
- "Skriv under för folkomröstning" petition button/modal referencing the real 2022 Katrineholm wind-power referendum.
- Turbines only render in the AR view when the weighted Outdoor Confidence Index (camera/AI sky detection, GPS accuracy, ambient light, compass stability, motion/gyro, WiFi/indoor signal) is in its "show" or "cautious" tier AND camera-based sky coverage is ≥15% of the frame; the "aim" tier (40-70%) shows a "Rikta kameran mot öppen himmel" banner instead of turbines, and the "hide" tier (<40%) or insufficient sky coverage shows the large, high-z-index "Gå utomhus" (go outside) message that always renders above all UI chrome (top/bottom bars, badges). This whole-screen OCI-based gate only ever applies while the (now-removed) ML sky-segmentation model would have been active (`useSkyDetection`'s `method === "ml"`, permanently `"disabled"`) — it never suppresses turbines or shows the "Gå utomhus"/aim banner on its own.
- Per-pixel occlusion (turbines hidden behind trees/buildings, not just a whole-turbine show/hide) is driven separately by `useSkyDetection`'s existing lightweight canvas-2D brightness/texture/saturation heuristic (the same one behind `outdoorConfidence`) — NOT by the old ML model. `isPointSky`/`getOcclusionGrid` now expose the real (EMA-smoothed) 12x8 sky/non-sky grid from that heuristic to `ARScene.tsx`'s existing per-fragment occlusion shader, so only the covered part of a turbine (e.g. below a treeline) is masked. This is coarser than the removed ML segmentation but uses no extra WebGL context and cannot reintroduce the freeze (see `docs/gotchas.md`). "Visa dolda"/showHiddenTurbines now **defaults to on** (see `VisualizationControls.tsx`): occluded parts render as sparse red half-transparent outlines instead of disappearing entirely, so a turbine partly behind a treeline never just silently vanishes; a manual "🎯 Dölj helt" toggle switches to fully hiding the occluded part instead.
- **(Updated July 2026 — critical bug fix, "turbines must never fully disappear"):** Real indoor/no-line-of-sight detection (`Home.tsx`'s `indoorsOrNoSight`, built directly on `useSkyDetection`'s always-running `sky.indoors`, gated on `ready` so it can't fire during startup) no longer forces turbine opacity to 0. `ARScene.tsx`'s `hideAll` prop now only dims every turbine/label/light/shadow to `INDOOR_DIM_FACTOR` (~45% opacity) with a gray/blue tint lerp (`INDOOR_TINT_COLOR`/`INDOOR_TINT_BLEND`) toward their `originalColors`, applied in `applyFinalOpacities` — same idea as the existing per-pixel occlusion shader (dim, don't hide), just at the whole-scene level instead of per-pixel. The full-screen "Du verkar vara inomhus..." overlay that used to replace the entire AR view is **removed**; the indoor state is now a small non-blocking `statusBanner` pill ("🏠 Skymd/inomhus – verken visas dämpade...") layered with the other status messages. Sound/dBA/infrasound/distance/bearing keep working throughout since they're computed from GPS alone, never gated on camera/sky state.
- **Priority order for final turbine opacity/tint (`applyFinalOpacities` in `ARScene.tsx`, and the parallel shadow-opacity calc in `animate()`):** `obj.forceVisible` (2s safety fallback) always wins first → full opacity, original colors, no matter what else is going on. Only if NOT force-visible does `hideAll` (indoor/no-sight) apply the dim+tint. Only if neither applies does the confidence-based `globalVisibilityFactor` apply, floored at `MIN_CONFIDENCE_VISIBILITY_FACTOR` (0.15) instead of ever reaching a hard 0. This ordering is intentional and load-bearing — see `docs/gotchas.md`'s "priority inversion" entry for the bug this fixes.
- A small always-visible `LineOfSightStatus` badge (top bar, next to the dBA/infrasound badges) shows "Fri sikt" (green) / "Delvis skymt" (yellow, sky coverage <60%) / "Ingen fri sikt" (red, `sky.indoors`) as a quick at-a-glance signal, independent of (and softer/more granular than) the `statusBanner` indoor pill above.
- `NearestTurbineArrow` (rendered whenever `ready`, positioned at screen edge z-50) always points toward the closest turbine by GPS bearing minus live compass heading, regardless of whether that turbine is on-screen, occluded, or the user is indoors — polls `useDeviceOrientation`'s `headingDegRef` on a plain interval (not a state subscription) to avoid re-rendering `Home.tsx` on every sensor tick. Hides itself only when the turbine is already visibly on-screen (within the AR camera's FOV) and the user is not indoors, fading into a "Du tittar mot närmaste verk" confirmation instead of vanishing; shows an indoor-specific instruction ("Vindkraftverket ligger åt detta håll – gå utomhus eller mot fri sikt för att se det.") instead of "Vrid mobilen åt detta håll" while `indoorsOrNoSight` is true.
- **Debug logging (added July 2026):** every pipeline step from the original bug report has an explicit, once-per-session `console.info("[AR][pipeline] ...")` breadcrumb — GPS OK / Compass OK / Loaded N turbines (`Home.tsx` and `ARScene.tsx`, since turbines can come from either the default set or a custom `/placera` placement) / Creating turbine #i / Anchor created / Model loaded / Renderer attached / Visible=true. The 2-second force-show safety net additionally logs a `console.warn("[AR][safety] ...")` when it triggers, naming how many turbines it force-showed and why.
- "🗺️ Placera vindkraftverken själv" (`/placera`), the free "kartverktyg": a pannable/zoomable satellite map (no camera/GPS/compass permissions needed), pre-loaded with the 8 real planned turbines nearest Katrineholm and an initial view wide enough to show Katrineholm/Ericsberg/Forssjö/Björkvik together. The user can click empty space to add a turbine, or tap an existing one to open a context menu (Flytta/Ta bort/Information/Avbryt) — "Flytta" enters a move-mode where the next tap commits the new position, replacing the old drag-to-move gesture so plain drag/pinch/double-tap always pan/zoom the map (Google-Maps-style) when nothing is selected. Each turbine is colored green/yellow/orange/red by a live per-turbine impact score; moving/adding/removing a turbine shows a brief "Beräknar påverkan…" indicator before the score/colors update. A "Visa Ericsbergs mark" toggle overlays a wide illustrative dark-fill/black-border polygon of Ericsberg's estate land, and a 🏰 "Ericsbergs slott" marker shows an owner-info popup. Also has a live overall 0–100 impact score (distance to households, sensitive zones, spacing), factoring in kommun-wide population/household data so placements near central Katrineholm/Forssjö correctly show non-zero household/noise/nature/culture/water impact instead of only scoring against the Ericsberg estate area, playful Swedish warnings, save/compare of multiple placements, and a "Se denna placering i AR" button that swaps the AR view's turbine set to the custom placement. A "✏️ Redigera gräns" mode turns the placement-area boundary into a real editable polygon — drag vertices, add new ones by tapping the map, remove with a ✕, save/reset the custom boundary, and export it as GeoJSON. An expanded "🐞 Felsökning" debug panel shows loaded data-layer counts and a full per-turbine breakdown (households within 1/2/3 km, distance to centrum, nearest nature/cultural/water zone, noise dBA, shadow-flicker score) for diagnosing any factor.
- After pressing the start screen's primary action, the user chooses between "📷 Starta AR" (the full camera/GPS/compass AR flow) or "🗺️ Öppna kartverktyg" (the permission-free map tool above) — both reachable directly from `PermissionGate.tsx`.
- Fully Swedish-language UI; installable as a PWA.
- AR placement stabilization ("prioritize stability over precision"): tracking is fused from GPS accuracy + compass heading stability into a tier (`useArTrackingStability.ts`). On degraded/lost tracking, the smoothed GPS position freezes at its last good value (rather than drifting) and a fixed banner reads "Svag positionering – använder senaste stabila läge"; unfreezing corrects smoothly over ~1-3s via the existing EMA rather than jumping. Turbines are never abruptly hidden — only after ~4s of sustained lost tracking do they fade toward a low-opacity floor over ~3s. Manual "Kalibrera om riktning"/"Kalibrera horisont" remain the user's explicit strong-reference reset. A hidden `SensorDebugPanel` (toggle in visualization controls) surfaces GPS accuracy, heading accuracy, pitch, yaw/heading, tracking tier, horizon offset, visible turbine count, and turbine hide-reasons for field diagnosis.
