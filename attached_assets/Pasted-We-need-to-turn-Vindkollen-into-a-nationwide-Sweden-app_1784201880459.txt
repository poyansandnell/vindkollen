We need to turn Vindkollen into a nationwide Sweden app with configurable local project modules.

1. Fix the Sweden project loading error

The native Sweden map currently shows:

“Could not load projects”
“The string did not match the expected pattern.”

This strongly suggests an invalid URL or fetch path under capacitor://localhost.

Please investigate and fix:

- Log the exact URL passed to fetch before every project-data request.
- Do not construct API URLs with undefined, empty or malformed base URLs.
- Detect Capacitor native runtime explicitly.
- For native builds, either:
  a) bundle the full project registry as local JSON/static TypeScript data, or
  b) use a valid absolute HTTPS production API URL.
- Never call fetch() with a relative API URL that resolves incorrectly under capacitor://localhost.
- Add a fallback bundled project list if the remote API is unavailable.
- Display “Loaded X projects” in temporary diagnostics.
- Log every loaded project ID, name, municipality and coordinate.
- Preserve the full web project dataset in dist-native.

The Sweden overview must remain usable even without a network connection if bundled project data is available.

2. Nationwide project architecture

Create a generic project registry.

Each project should contain:

- id
- name
- municipality
- region/county
- latitude
- longitude
- turbineCount
- turbines
- boundary polygon
- project description
- source documents
- optional local campaign configuration
- optional contact details
- AR configuration

The app must not contain hardcoded Ericsberg-specific routing or labels.

When a user selects a project, pass selectedProjectId to the generic editor and AR view.

3. Start flow

The user should be able to:

- View all Sweden
- Select municipality/project
- Search by place or project name
- Select a marker on the Sweden map
- Zoom into a project
- Open that project’s generic detail/editor/AR view

Add a selector such as:

“Välj ort eller projekt”

Initially include Katrineholm, but make it data-driven so additional municipalities can be added without changing the UI logic.

4. Local campaign modules

Do not show the Katrineholm referendum form globally.

Add optional project-level campaign configuration.

Example:

campaign: {
  enabled: true,
  type: "referendum-interest",
  title: "...",
  description: "...",
  municipality: "Katrineholm"
}

Only show the referendum/interest form when the selected project has campaign.enabled = true.

For all other projects, hide the campaign CTA completely.
