- hub

The model must still be lightweight enough to run smoothly on iPhone Safari.

2. Rotor blades
The blades are currently rotating around the wrong axis.
Rotate them around the turbine hub axis, like a real wind turbine.
The rotor should face the user correctly and rotate realistically at approximately 8–15 RPM.
Only the rotor/blade group should rotate, not the whole turbine.

3. Aviation warning lights
The red aviation warning lights must blink like real obstacle lights.
Use synchronized blinking:
- 0.2 seconds ON
- 1.8 seconds OFF
- all turbines blink simultaneously
At night, the red light should also create a subtle glow around the nacelle.

4. Stable AR positioning
The turbines still drift even when the phone is standing still.
Improve stabilization by:
- using DeviceOrientationEvent carefully
- applying smoothing / low-pass filtering
- ignoring very small sensor movements
- reducing jitter
- locking the horizon after calibration
The turbines should appear fixed in the landscape, not floating around.

5. Horizon and vertical position
The turbines should remain attached to the real horizon.
When the phone tilts up or down, they should move naturally in perspective, but not follow the screen like stickers.
Add or improve the button:
“Kalibrera horisont”
When clicked, store the current pitch/beta angle as the horizon reference.
